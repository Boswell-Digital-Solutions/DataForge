## 📝 Maintenance Guidelines

### Adding New Documentation

**When creating new documentation, follow these guidelines:**

1. **Session Summaries** → `docs/sessions/`
   - File naming: `SESSION_SUMMARY_YYYY-MM-DD.md`
   - Specialized sessions → appropriate subdirectory (e.g., `docs/sessions/typescript/`)

2. **Phase Completion Reports** → `docs/archive/phases/`
   - File naming: `PHASE_X.Y_COMPLETE_SUMMARY.md`
   - Keep current phase summary in component root until next phase begins

3. **Product Documentation** → `docs/{product_name}/`
   - Feature-specific docs in product subdirectory
   - General guides → `docs/guides/`

4. **Implementation Guides** → `docs/guides/`
   - Setup guides, integration guides, howtos

5. **Context Files** → `docs/context/`
   - Planning documents, context for specific integrations

6. **Archive Old Docs:**
   - When documentation becomes outdated, move to `docs/archive/`
   - Keep reference value, remove from active docs

### Root Directory Rules

**Only these file types belong in root:**
- ✅ README.md (main documentation)
- ✅ INDEX.md or similar navigation files
- ✅ CHANGELOG.md (version history)
- ✅ QUICK_REFERENCE.md (frequently accessed)
- ✅ Recent consolidation reports (temporary, archive after 30 days)
- ❌ Session summaries (→ docs/sessions/)
- ❌ Phase reports (→ docs/archive/phases/)
- ❌ Product-specific docs (→ docs/{product}/)
- ❌ Implementation guides (→ docs/guides/)
